# 한국어 번역 가이드라인

## 번역 원칙

### 1. 기술 용어 처리 (병기 방식)

**첫 등장 시:**
- 한국어(영어) 형식으로 병기
- 예: `작업 증명(Proof of Work)`
- 약어의 경우: `최대 추출 가능 가치(Maximal Extractable Value, MEV)`

**이후 등장:**
- 영어 원문만 사용
- 예: `Proof of Work`, `MEV`

**병기가 필요 없는 경우:**
- 일반적으로 통용되는 용어: Bitcoin → 비트코인, Ethereum → 이더리움
- 고유명사: Solana, Uniswap, Aave 등

### 2. 마크다운 구조

- 원본과 동일한 헤딩 구조 유지
- 코드 블록, 링크, 이미지는 원문 그대로
- 표(table) 형식 유지

### 3. 번역 톤

- 교육적이고 명확한 어조
- 전문 용어는 정확하게, 설명은 이해하기 쉽게
- 원문의 의도와 뉘앙스 보존

### 4. 숫자 및 단위

- 숫자는 원문 그대로: 21 million
- 단위도 영문 유지: BTC, ETH, gas, gwei

### 5. 코드 및 기술 예시

- 코드는 절대 번역하지 않음
- 코드 주석만 번역 가능 (문맥상 필요시)
- 주소, 해시값 등은 원문 유지

## 워크플로우

### 각 챕터 번역 시:

1. **참고자료 확인** - 해당 챕터 관련 공식 문서 확인
2. **본문 번역** - Opus 모델로 고품질 번역
3. **검증** - 용어 일관성, 누락 확인
4. **Issue 기록** - 의문점이나 원문 오류 기록

## 파일 명명 규칙

- 번역본: `{원본 파일명}.md`
- 이슈: `{원본 파일명}_issues.md`

예시:
- `Chapters/_preface.md` → `ko/Chapters/_preface.md` + `ko/Chapters/_preface_issues.md`

## 품질 기준

- 기술적 정확성 최우선
- 한국 암호화폐 커뮤니티의 통용 표현 존중
- 원문의 구조와 흐름 유지
- 자연스러운 한국어 문장
